🔄 数据处理流程总览

Bulk 转录组 / 微阵列处理流程

  1. GEO 数据下载：使用 run_download_gse.sh 一键下载并清洗。脚本自动处理芯片/RNA-seq 数据，包括 GPL 注释、探针映射、SYMBOL 聚合、生成质控报告。
  2. 自动探针/ID 映射：脚本内部自动完成探针 → SYMBOL 映射，同一基因多探针默认取均值（--symbol-agg-method mean），可改为 max_row_sum。
  3. 表达量预处理：
    • RNA-seq：默认输出 TPM，可加 --no-tpm 输出原始 Count
    • 芯片：自动检测 log2 尺度并在报告中标注
  4. 分组信息提取：使用 run_extract_groups.sh 从 pdata.csv 中提取分组，支持自动检测、手动指定列、分组映射/重命名
  5. 格式输出：下载工具输出 *_cleaned_expression.csv + *_pdata.csv，分组工具输出 groups.csv，最终整理为标准 expression.csv / group.csv 进入分析模块
📌 输出文件结构（run_download_gse.sh）： 
output/GSEXXXXX/
├── GSEXXXXX_cleaned_expression.csv   # 表达矩阵（基因 × 样本；芯片/TPM/FPKM）
├── GSEXXXXX_counts.csv               # Count 矩阵（仅 RNA-seq Count 时生成）
├── GSEXXXXX_pdata.csv                # 样本元信息（用于 run_extract_groups.sh）
├── GSEXXXXX_fdata.csv                # 探针/基因注释
├── GSEXXXXX_data.qs2                 # 完整数据（qs2 格式，含 geo 对象）
└── GSEXXXXX_report.txt               # 数据质量报告

单细胞 scRNA-seq 处理流程

  1. 数据下载：从 GEO 下载 10X Genomics 格式文件（barcodes.tsv / features.tsv / matrix.mtx）或 h5 文件
  2. 创建 Seurat 对象：使用 Read10X() 或 Read10X_h5() 加载数据，创建 CreateSeuratObject()
  3. 质控过滤：
    • 过滤低质量细胞（nFeature_RNA、nCount_RNA、percent.mt）
    • 过滤双联体（DoubletFinder 等）
    • 记录过滤前后细胞数量
  4. 样本合并与整合：多样本需合并后做批次校正（如 Harmony / CCA / RPCA），记录整合方法和参数
  5. 分组标注：根据项目需求标注样本分组（疾病/对照），整理为元数据
⚠️ 单细胞数据注意：单细胞分析流程与 bulk 差异较大，通常不输出标准 CSV 格式，而是保存为 .qs2（Seurat 对象，使用 qs2 包序列化）供后续分析模块使用。样本分组决策需在数据反馈阶段确认。

📋 输入数据格式标准 必须遵守

⚠️ 核心规则： 所有输入文件统一使用 CSV 格式；数值列必须是 num 或 int 类型；分类列必须是 chr 类型；不允许有 NA 值。

📊 expression.csv — 表达矩阵

项目要求
行基因（每行一个基因）
列样本（每列一个样本）
第一列列名为 SYMBOL（大写），内容为基因 Symbol
数据类型表达值为 num 或 int，可以存在负值
缺失值不允许有 NA
基因物种人、小鼠、大鼠（暂时只适配这三种）

🧹 Step 2：分组信息提取（使用 run_extract_groups.sh）

📌 工具位置：/media/share/secure/gse_download_and_cleaning/run_extract_groups.sh
推荐工作流（三步）： 
# ─── 第 1 步：浏览可用分组列（只看不写）───
run_extract_groups.sh -i output/GSE65204/GSE65204_pdata.csv
# → 自动检测所有可能的分组列，打印每列的唯一值和样本数

# ─── 第 2 步：选定列 + 映射/重命名分组（确认结果正确再输出）───
run_extract_groups.sh -i output/GSE65204/GSE65204_pdata.csv \
    -c "asthma:ch1" \
    --map "TRUE=CA,FALSE=Control"
# → 屏幕预览映射结果，不加 -o 不会写文件

# ─── 第 3 步：确认无误后输出到文件 ───
run_extract_groups.sh -i output/GSE65204/GSE65204_pdata.csv \
    -c "asthma:ch1" \
    --map "TRUE=CA,FALSE=Control" \
    -o output/GSE65204/groups.csv
📌 参数说明：
参数说明默认值
-i / --inputpdata.csv 文件路径—
-c / --column指定分组列名（如 "disease:ch1"）自动检测
--map分组映射："原值1=新值1,原值2=新值2"—
--mapped-only仅保留被映射的样本TRUE
-o / --output输出文件（不指定则仅预览）—
--show-values显示每个分组列的唯一值FALSE
--max-unique分组列最大唯一值数量10
⚠️ 注意：run_download_gse.sh 已自动完成探针→SYMBOL 映射、log2 检测等预处理，无需手动处理。这一步只需提取分组信息。

🔄 Step 3：格式转换为标准输入

下载工具输出的 *_cleaned_expression.csv 和 groups.csv 需转换为标准输入格式：

library(data.table)

# 1. expression.csv — 整理列名，确保第一列为 SYMBOL
expr <- fread("output/GSEXXXXX/GSEXXXXX_cleaned_expression.csv", data.table = FALSE)
colnames(expr)[1] <- "SYMBOL"
fwrite(expr, "expression.csv")

# 2. group.csv — 从 groups.csv 整理为标准两列格式
groups <- fread("output/GSEXXXXX/groups.csv", data.table = FALSE)
# 确保只有 sample + group 两列
group_std <- data.frame(
    sample = groups$geo_accession,    # 或 groups$sample_id，按实际列名
    group  = groups$group             # run_extract_groups.sh 输出的分组列
)
fwrite(group_std, "group.csv")

# 3. gene.csv（如有特定基因集）
gene_df <- data.frame(SYMBOL = target_genes)
fwrite(gene_df, "gene.csv")

# ─── 交叉校验 ───
expr_samples  <- colnames(expr)[-1]
group_samples <- group_std$sample
stopifnot("sample 不匹配" = length(setdiff(group_samples, expr_samples)) == 0)
⚠️ 输出前必须检查：
  • 所有数值列确认为 numeric / integer，不能是字符串
  • 所有分类列确认为 character
  • 检查并移除所有 NA 值
  • 检查 group.csv 的 sample 列与 expression.csv 列名完全匹配

👥 Step 4：分组决策记录

数据反馈文件（Excel）通常会给出分组建议，需要在此阶段确认和记录：

决策项示例说明
哪些样本作为疾病组仅哮喘样本 / 包含合并症样本避免混杂因素干扰，需客户确认
哪些样本作为对照组健康对照 / 正常组织确保组间有可比性
排除的样本质量不达标 / 不相关亚型记录排除原因
多数据集验证训练集 GSE_A + 验证集 GSE_B明确各数据集的用途
✅ 最佳实践：在初步处理日志中详细记录分组决策的依据和最终方案，便于后续追溯。如果数据反馈文件中有多个分组方案可选，务必等待客户确认后再处理。

📊 DEGs 模块 — 差异表达分析

# ---- 基础用法 ----
bash run.sh \
    --module DEGs \
    --expression expression.csv \
    --group group.csv \
    --method DESeq2 \
    --pvalue 0.05 \
    --log2fc 1 \
    --output result/DEGs/

# ---- 仅重新作图（不重跑计算）----
bash run.sh \
    --module DEGs \
    --plot-only \
    --output result/DEGs/

# ---- 参数说明 ----
# --method     : DESeq2 / edgeR / limma（根据数据类型选择）
# --pvalue     : 显著性阈值
# --log2fc     : log2 倍数变化阈值
# --plot-only  : 仅重新生成图表

✅ 数据校验清单

✨ 校验原则：所有输入文件在进入分析模块前必须通过校验。校验结果输出为结构化格式（通过 / 失败 + 错误原因 + 修正建议）。

通用校验项

📄
文件格式
必须是 CSV 格式，UTF-8 编码（无 BOM）
🚫
缺失值
不允许任何 NA / NaN / 空值
🔢
数据类型
数值列为 num/int，分类列为 chr
📝
列名规范
SYMBOL 大写，sample/group 小写

文件特定校验项

文件校验项规则自动修正
expression.csv 第一列名 必须为 SYMBOL —
基因名格式 符合人/鼠/大鼠基因命名规范 —
重复基因 不允许重复的 SYMBOL 取均值（默认）或行和最大值
log2 尺度检测 最大值 > 25 提示需要 log2 转换 log2(x+1)
group.csv 列名 必须包含 sample 和 group —
样本匹配 sample 值必须与 expression.csv 列名一致 —
survival.csv OS 值域 OS 列只能是 0 或 1 —
OS.time 值域 OS.time 必须 > 0 —
riskscore.csv group 值域 group 列只能是 "High" 或 "Low" —

校验脚本示例

# ---- R: 数据校验函数 ----
validate_input <- function(file_path, file_type = c("expression", "group", "gene",
                                                    "survival", "riskscore")) {
  file_type <- match.arg(file_type)
  dat <- data.table::fread(file_path, data.table = FALSE)
  errors <- character()
  warnings <- character()

  # ---- 通用校验 ----
  if (any(is.na(dat))) {
    na_cols <- colnames(dat)[colSums(is.na(dat)) > 0]
    errors <- c(errors, paste0("发现 NA 值，涉及列: ", paste(na_cols, collapse = ", ")))
  }

  # ---- expression 专用 ----
  if (file_type == "expression") {
    if (colnames(dat)[1] != "SYMBOL")
      errors <- c(errors, "第一列名必须为 SYMBOL")
    if (any(duplicated(dat$SYMBOL)))
      warnings <- c(warnings, paste0("存在 ", sum(duplicated(dat$SYMBOL)),
                                      " 个重复基因，建议取均值聚合"))
    max_val <- max(dat[, -1], na.rm = TRUE)
    if (max_val > 25)
      warnings <- c(warnings, paste0("最大表达值 = ", round(max_val, 2),
                                      "，可能需要 log2 转换"))
  }

  # ---- survival 专用 ----
  if (file_type == "survival") {
    if (!all(dat$OS %in% c(0, 1)))
      errors <- c(errors, "OS 列只能包含 0 或 1")
    if (any(dat$OS.time <= 0))
      errors <- c(errors, "OS.time 必须 > 0")
  }

  # ---- riskscore 专用 ----
  if (file_type == "riskscore") {
    if (!all(dat$group %in% c("High", "Low")))
      errors <- c(errors, 'group 列只能是 "High" 或 "Low"')
  }

  # ---- 输出结果 ----
  list(
    status   = if (length(errors) == 0) "PASS" else "FAIL",
    errors   = errors,
    warnings = warnings,
    summary  = paste0("行数: ", nrow(dat), " | 列数: ", ncol(dat))
  )
}

# ---- 使用示例 ----
result <- validate_input("expression.csv", "expression")
cat("[", result$status, "] ", result$summary, "\n")
if (length(result$errors) > 0)   cat("  错误: ", paste(result$errors, collapse = "\n         "), "\n")
if (length(result$warnings) > 0) cat("  警告: ", paste(result$warnings, collapse = "\n         "), "\n")

⚠️ 常见问题与注意事项

📐 log2 转换判断

场景最大值操作
微阵列（已标准化）< 25无需转换，已在 log2 尺度
微阵列（原始信号值）> 25执行 log2(x + 1)
RNA-seq count通常很大保留原始 count，由 DESeq2/edgeR 处理
RNA-seq TPM/FPKM视情况如用于机器学习，可 log2(x + 1)
⚠️ 关键警告：
  • DESeq2 禁止输入已 log 转换的数据，只接受原始 count
  • 微阵列数据通常已由平台标准化过，先确认再决定是否转换

🔬 探针/ID 映射注意事项

  • 同一基因多探针：默认取均值（mean），可通过 --symbol-agg-method max_row_sum 改为取行和最大值
  • 无法映射的探针：直接丢弃（不保留无 Symbol 的探针）
  • GPL 注释文件字段名：不同平台的列名可能不同（如 "Gene Symbol"、"gene_assignment"、"Symbol"），需手动确认
  • Ensembl → Symbol：使用 org.Hs.eg.db（人）/ org.Mm.eg.db（鼠）/ org.Rn.eg.db（大鼠）

🔄 批次效应处理

  • 多数据集合并：使用 run_combine.sh 一键完成 ComBat 批次校正，自动生成校正前后 PCA/箱线图对比
  • RNA-seq + 微阵列混合：通常不建议直接合并，建议作为独立验证集使用
⚠️ 批次校正注意：
  • 校正前必须确保各批次的分组比例相近，避免批次与分组混淆
  • 校正后检查 PCA 图（04.pca_after_dataset.*），确认批次效应已消除但组间差异保留
  • 批次校正参数和方法已自动写入日志和报告

👥 样本名匹配

  • 常见问题：expression.csv 列名与 group.csv 的 sample 列不一致
  • 原因：GEO 下载后列名可能带版本号（.1）、空格或特殊字符
  • 检查方法： 
    # 快速检查样本名匹配
expr_samples <- colnames(fread("expression.csv", nrows = 0))[-1]
group_samples <- fread("group.csv")$sample

# 在 expression 但不在 group 中的样本
setdiff(expr_samples, group_samples)

# 在 group 但不在 expression 中的样本
setdiff(group_samples, expr_samples)
  • 修正方法：统一使用 GEO accession（如 GSM1589502）作为样本名

🔬 单细胞数据特有问题

  • 样本分组决策：单细胞数据集中可能包含多种亚型（如哮喘 + 合并感染），需在数据反馈阶段明确哪些样本纳入分析
  • 数据格式：单细胞不输出 expression.csv 格式，而是保存为 Seurat 对象（.qs2 格式，使用 qs2::qs_save()）
  • 质控阈值：需根据数据集实际情况调整（nFeature_RNA、percent.mt 等），不能使用固定阈值
  • 整合方法选择：
    方法适用场景速度
    Harmony大多数场景，性能均衡快
    CCA批次间差异大慢
    RPCA大数据集，内存友好中
