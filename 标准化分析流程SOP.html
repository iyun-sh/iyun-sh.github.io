📋 1.5 通用输入文件格式规范

以下是各模块共用的标准文件格式定义。各模块输入表中的「格式」列引用此处编号。

编号文件名模式格式列要求说明
F1expression.csv / *.dat.csvCSV, UTF-8第一列 SYMBOL（基因名），其余列为样本，值为数值型表达量行=基因，列=样本。支持 raw counts / log2 / TPM
F2group.csvCSV, UTF-8sample, groupsample 值必须与表达矩阵列名精确匹配
F3gene.csv / genes.csvCSV, UTF-8SYMBOL（或 gene），每行一个基因名通常来自 DEGs / Venn / MachineLearn 输出
F4survival.csvCSV, UTF-8sample, OS.time(天数), OS(0/1)也支持 DFS.time/DFS、PFS.time/PFS 等终点
F5riskscore.csv / rs_dat.csvCSV, UTF-8sample, riskscore, riskgroup来自 Prognosis 模块输出
F6config.iniINI 格式[section] + key = value各模块的配置文件，指定输入路径和参数
F7gse_list.txt纯文本每行一个 GSE ID（如 GSE126124）批量下载模式使用
F8MAF 数据TCGA MAF标准 MAF 列（Hugo_Symbol, Variant_Classification 等）TMB 模块自动从 TCGA 下载，通过 cancer_type 指定
F9GWAS 汇总统计TSV/TXTSNP, A1, A2, FREQ, BETA, SE, PMR / TSG 模块使用的标准 GWAS 汇总格式
📌 通用约定：所有 CSV 文件使用英文逗号分隔、UTF-8 编码、首行为列名。基因名统一使用 HGNC Official Symbol 格式。

9. 表达验证 (Expression_verify) iyunlyl
Wilcoxon 秩和检验 + ROC/AUC 跨数据集验证候选基因

操作步骤

📋 数据来源说明
  • -t 训练集：使用 Combine 输出的 combat.dat.csv（全基因表达矩阵），模块会自动过滤至 -G 指定的候选基因
  • -g 训练集分组：使用 Combine 输出的 combat.group.csv
  • -v 验证集：使用 GEO 模块单独下载的外部数据集（如 GSE126124），不需要经过 Combine 合并
  • -G 候选基因：来自 Venn 交集输出（01.venn_intersect.csv）或 MachineLearn 单算法输出
/media/share/secure/Expression_verify/run_expression_verify.sh \ -t train_expr.csv \ -g train_group.csv \ -v val1_expr.csv,val2_expr.csv \ -V val1_group.csv,val2_group.csv \ -G genes.csv \ -n TrainSet,GSE126124,GSE48958 \ -o /media/share/Project/myproject/07_Expression_verify

关键参数

参数说明
-t, --train-expr训练集表达矩阵 CSV（第一列 SYMBOL，通常用 Combine 输出的 combat.dat.csv）
-g, --train-group训练集分组 CSV（sample, group 两列，样本名必须与 -t 列名一致）
-v, --val-expr验证集表达矩阵（逗号分隔多个文件），数量和顺序必须与 -V 一一对应
-V, --val-group验证集分组文件（逗号分隔，个数必须等于 -v 中的文件数）

现象可能原因解决方案

log2 变换后数据异常 数据已是 log 尺度但被重复变换 GSE 下载模块自动检测 log2 状态，通常无需手动干预
