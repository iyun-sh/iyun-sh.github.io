🧬 生信流程执行员培训手册

面向标准化分析流程的新人培训方案。不教算法原理，只教标准操作。
覆盖 3 个方向 × 7 天速成，配套测试数据与考核标准。

📖 0. 概述

培训核心思路

不教原理，只教操作；不教分析，只教执行。

新人只需做到三件事：

  1. 会按 SOP 准备输入文件、调用入口脚本
  2. 会看日志和报告判断流程是否成功
  3. 会按质控标准判定结果是否合格

培训方向

根据模块类型和技能要求，分为 3 个方向：

# 方向 负责模块 核心技能 培训周期
1 数据获取 gse_download、combine GEO 数据下载、格式校验、数据合并 3 天
2 标准分析 degs、enrichment、venn、ml、prognosis、nomogram、exprverify、wgcna、ppi、friends、immune、chrplot、depth2、tmb、msi 输入文件准备（F1-F5）、批量执行、报告核查 5 天
3 因果推断 mr、tsg GWAS 数据处理（F9）、MAF 数据（F8）、特殊结果解读 2 天
💡 建议路径：所有新人从「数据获取」Day 1-3 开始 → 考核通过后进入「标准分析」Day 1-5 → 有需要再扩展「因果推断」。

方向定位

你是流程执行者，不是算法研发。核心职责：

  • 按标准操作准备输入文件、运行脚本
  • 查看日志和报告，判断是否成功
  • 按质控标准检查结果，不合格的标记处理
  • 结果归档交付，记录运行日志
  • 不擅自修改参数、不修改脚本代码

🚨 1. 红线纪律（必须遵守）

🚫 以下行为是绝对红线，违反将严肃处理：
# 红线行为 后果

⚠️ 核心原则：只照做，不创新。参数、路径、文件名严格按手册。不确定就问。

🏗️ 3. 系统架构认知

目录结构

# 脚本目录（受保护，不可修改） /media/share/secure/ ← R 脚本源码（加密） /media/share/secure/run_*.sh ← 入口脚本 # 项目输出目录 /media/share/Project/ └── 项目名/ ← 每个项目一个目录 ├── expression.csv ← 输入文件 ├── group.csv ├── gene.csv ├── survival.csv ├── riskscore.csv ├── output/ ← 分析输出 │ ├── DEGs/ │ ├── ML/ │ ├── report/ │ └── ... └── log/ ← 运行日志

统一调用模式

所有 20 个模块遵循相同的调用模式：

# 1. 进入项目目录 cd /media/share/Project/项目名/ # 2. 准备输入文件（按格式要求） # 详见 SOP「1.5 通用输入文件格式规范」→ F1-F9 # 3. 调用入口脚本 bash /media/share/secure/run_模块名.sh [参数] # 4. 检查输出 ls -la output/模块名/

核心概念

概念说明
入口脚本 所有模块通过 run_*.sh 入口脚本调用，新人不需要接触 R 代码
代码保护 R 源码在 /media/share/secure/ 下加密保护，只能通过入口脚本间接调用
输入文件 标准化 CSV/TXT 格式，9 种格式编号 F1-F9（详见 SOP 第1.5节）
config.ini 部分模块使用 INI 配置文件控制参数，由负责人配置，执行员不修改

Day 1 GEO 数据下载

目标：学会使用 run_download_gse.sh 下载 GEO 数据集，理解输入输出。

  • 能独立编写 gse_list.txt
  • 能运行下载脚本并等待完成

2.2 数据格式校验

对每个下载的数据集执行以下检查：

#检查项命令合格标准
1 expression.csv 首列 head -1 expression.csv 第一列为 SYMBOL

✅ 5. 质控标准

每个模块运行完毕后，必须按以下标准逐项核查。不合格禁止交付。

5.1 通用质控项（所有模块）

#检查项方法合格标准不合格处理
1 脚本退出状态 查看日志末尾 无 Error、无 Traceback、显示完成 记录错误信息，重跑或上报
2 输出目录存在 ls output/模块名/ 目录存在且非空 检查输入文件是否正确
3 关键文件完整 ls -la output/模块名/*.csv *.pdf *.png 预期文件全部存在，大小 > 0 对照 SOP 检查遗漏原因
4 图片可打开 下载查看 图片清晰、标注完整、无截断 重跑
5 报告完整 浏览器打开 HTML 报告 所有节均正常渲染，无空白节 重跑

5.2 模块专项质控

DEGs 差异分析

检查项合格标准
差异基因数量上调 + 下调基因总数 ≥ 50（阈值 |logFC| > 1, adj.P < 0.05）
火山图左右对称散布，有显著标注点
热图分组有明显差异色块

ML 机器学习筛选

检查项合格标准
riskscore.csv 输出存在且格式符合 F5，riskgroup 只有 High/Low
筛选基因数通常 5-30 个，极端值需确认
AUC 值训练集 AUC ≥ 0.6

Enrichment 功能富集

检查项合格标准
GO 条目数BP/CC/MF 各有结果（≥ 5 条显著）
KEGG 通路数≥ 3 条显著通路
气泡图有明确可读的通路名称

Prognosis 预后分析

检查项合格标准
KM 曲线高低风险组有明显分离，P < 0.05
ROC 曲线1/3/5 年 AUC 均有值

Immune 免疫浸润

检查项合格标准
免疫细胞比例各样本比例之和 ≈ 1（CIBERSORT）
差异箱线图至少部分细胞类型有显著差异

WGCNA 权重共表达网络

检查项合格标准
模块数量≥ 3 个非灰色模块
软阈值R² ≥ 0.8 对应的 power 值存在

TMB 肿瘤突变负荷

检查项合格标准
TMB 计算值数值范围合理（通常 0.1-100 mutations/Mb）
瀑布图前 N 基因突变频率可视化正常

⚠️ 6. 常见错误速查

遇到错误时先在此表查找，复制粘贴对应解决方案。解决不了的直接上报，不要自己乱改。

# 错误现象 原因 解决方案
1 No space left on device 磁盘空间不足 运行 df -h 确认 → 清理临时文件或通知管理员
2 Permission denied 文件权限不足 检查文件所有者，联系管理员授权
3 cannot open file 'expression.csv' 输入文件不存在或路径错误 确认 pwd 在正确目录，ls 检查文件存在
4 subscript out of bounds 数据行列数不匹配 检查 expression.csv 和 group.csv 的样本名是否一致
5 object 'xxx' not found 前置步骤未完成或输入文件缺列 按 SOP 检查输入文件格式（F1-F9 列名要求）
6 脚本运行很久没结束 正常（大数据耗时）或卡死 用 ps aux | grep R 看进程状态；CPU 占用 0% 持续 10 分钟 = 卡死 → kill 后重跑
7 Error in library(xxx) R 包未安装 不要自行安装，通知管理员处理
8 输出图片空白/截断 数据量太少或格式异常 检查输入数据行数（wc -l），确认不低于最低要求
9 GEO download failed 网络问题或 GEO 服务器不可用 等待 10 分钟后重跑；如持续失败，换网络或上报
10 group.csv 分组数 ≠ 2 DEGs/depth2 等需恰好 2 组 检查 group.csv 的 group 列去重后是否恰好 2 个值
🚫 三不原则：看不懂的错误不猜、自己解不了的不试、涉及脚本/配置的不改 → 截图+日志上报。

📝 7. 考核标准

7.1 数据获取考核（Day 3 完成后）

#考核项方式通过标准
1 独立下载数据 给定 3 个 GSE 编号，独立完成下载 全部下载成功，md5/文件完整性通过
2 格式校验 给定一份故意有错的 expression.csv 能发现全部格式问题（缺列、空值、格式错误）
3 数据合并 合并两个数据集 合并结果样本数正确、无 NA
4 日志判断 给 3 段日志片段 准确判断成功/失败并说明原因

7.2 标准分析考核（Day 5 完成后）

#考核项方式通过标准
1 完整流水线 从 expression.csv 开始，跑完 DEGs → enrichment → ML → prognosis 全链路 每个模块输出正确，质控全部通过
2 输入判断 给定一个模块名，说出需要哪些输入文件 15 个模块全部正确
3 错误处理 模拟 3 种常见错误场景 能按速查表独立解决
4 质控判定 给一份模块输出，按质控表检查 逐项核对，正确标记合格/不合格
5 归档交付 整理输出目录结构 文件完整、命名规范、日志保留

7.3 因果推断考核（Day 2 完成后）

#考核项方式通过标准
1 GWAS 数据识别 判断给定文件是否符合 F9 格式 能识别缺失列和格式错误
2 MR 运行 独立运行一次 MR 分析 输出因果推断结果，能解读方向
3 TSG 运行 独立运行一次 TSG 分析 输出结果正常

7.4 考核通过标准

✅ 通过 = 所有考核项全部达标
  • 流程跑完无错
  • 质控全部合格
  • 归档规范、无遗漏文件
  • 遇到错误能按速查表处理或正确上报
⚠️ 独立作业后前 3 次任务：由负责人复核质控结果，确认无误后方可独立作业。

📋 附录 A. 数据集反馈表规范

数据获取完成数据下载清洗后，必须输出一份「数据集反馈表」（Excel），用于记录获取的数据集信息、样本构成和分组方案。这是数据交付的标准产物之一。

🚫 没有反馈表 = 交付不完整。每个项目下载完毕后必须填写。

A.1 反馈表结构

每份反馈表包含以下 sheet：

Sheet内容必填
总览（Sheet1） 所有数据集的概要信息表 ✅ 是
各数据集详情（按 GSE 编号命名） 每个数据集一个 sheet，列出样本级详情 ✅ 是

A.2 总览表列定义

#列名说明示例
1编号GEO 数据集编号GSE65204
2疾病/组织类型研究对象儿童哮喘/鼻气道上皮细胞
3测序技术/平台技术平台信息Microarray (GPL14550)
4主要样本构成各组样本数量36 例 CA + 33 例 Control
5样本取材类型组织来源鼻上皮细胞
6细胞/基因数量单细胞填细胞总数；Bulk 填基因数（可选）61,596 细胞 / 20,186 基因
7备注分组说明、特殊处理建议推荐哮喘组 vs Control 组

A.3 数据集详情 Sheet 列定义

每个 GSE 编号对应一个详情 sheet，包含样本级信息：

#列名说明示例
1sampleGEO 样本编号GSM8722221
2group分组标签Control
3source_name样本来源描述Nasal airway epithelial cells
4characteristics原始特征字段asthma: TRUE

A.4 填写流程

  1. 下载完成后：对每个 GSE，从 GEO 页面获取数据集元信息填入总览表
  2. 分组提取后：运行 extract_groups 脚本，将输出的 group.csv 内容复制到详情 Sheet
  3. 格式校验后：将 expression.csv 行数（基因数）填入总览表
  4. 确认分组：在备注列写清楚哪些样本作为实验组、哪些作为对照组
  5. 提交复核：反馈表随数据文件一起交付
