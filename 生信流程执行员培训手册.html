📖 0. 概述

培训核心思路

不教原理，只教操作；不教分析，只教执行。

新人只需做到三件事：

  1. 会按 SOP 准备输入文件、调用入口脚本
  2. 会看日志和报告判断流程是否成功
  3. 会按质控标准判定结果是否合格

培训方向

根据模块类型和技能要求，分为 3 个方向：

# 方向 负责模块 核心技能 培训周期
1 数据获取 gse_download、combine GEO 数据下载、格式校验、数据合并 3 天
2 标准分析 degs、enrichment、venn、ml、prognosis、nomogram、exprverify、wgcna、ppi、friends、immune、chrplot、depth2、tmb、msi 输入文件准备（F1-F5）、批量执行、报告核查 5 天
3 因果推断 mr、tsg GWAS 数据处理（F9）、MAF 数据（F8）、特殊结果解读 2 天
💡 建议路径：所有新人从「数据获取」Day 1-3 开始 → 考核通过后进入「标准分析」Day 1-5 → 有需要再扩展「因果推断」。

方向定位

你是流程执行者，不是算法研发。核心职责：

  • 按标准操作准备输入文件、运行脚本
  • 查看日志和报告，判断是否成功
  • 按质控标准检查结果，不合格的标记处理
  • 结果归档交付，记录运行日志

1.3 分组提取

下载完成后，使用 run_extract_groups.sh 从 *_pdata.csv 中提取分组信息。这是一个命令行参数式工具（非交互式菜单），建议分三步操作：

参数说明
参数说明默认值
-f / --filepdata.csv 文件路径（必需）—
-c / --column指定分组列名（如 "disease:ch1"）—
-a / --auto-detect自动检测分组列TRUE
--no-auto-detect禁用自动检测，仅使用 -c 指定的列—
--map分组映射，格式 "原值=新值"，可重复指定多个（如 --map "HC=Control" --map "NASH=Disease"）—
--mapped-only仅保留被 --map 映射过的样本，未被映射到的分组会被丢弃。例如 pdata 有 3 个分组 A/B/C，你只 --map "A=Tumor" --map "B=Normal"，则 C 组样本会被排除。设为 FALSE 则保留所有样本（未映射的保持原值）TRUE
-o / --output输出文件路径（不指定则仅预览，不写文件）—
--output-cols输出列：all / sample_id / groups_onlyall
--show-values显示每个分组列的唯一值FALSE
--max-unique分组列最大唯一值数量，超过的列被跳过10
--max-values显示唯一值的最大数量20
--min-common-len最小公共部分长度2
-q / --quiet静默模式，不打印详细信息FALSE
-h / --help显示帮助信息—
操作步骤详解（三步法）

第一步：浏览可用分组列（只看不写）

只传 -f，脚本自动检测所有可能的分组列，打印每列的唯一值和样本数：

# 浏览 pdata 中的分组候选列 /media/share/secure/gse_download_and_cleaning/run_extract_groups.sh \ -f 01_GSE/GSE31210/GSE31210_pdata.csv # 输出示例： # === Candidate group columns === # [disease_state] (2 unique values) # "lung adenocarcinoma" (n=206) # "adjacent normal" (n=20) # [tissue] (1 unique values) # "lung" (n=226)

根据输出，确定要用哪一列（本例选 disease_state），以及原始值需要映射成什么。

第二步：指定列 + 映射，预览结果（仍不写文件）

加上 -c 和 --map，但不加 -o，脚本只在屏幕上打印映射结果，方便确认。--map 可重复指定多个：

# 预览映射结果（不加 -o，不会写文件） /media/share/secure/gse_download_and_cleaning/run_extract_groups.sh \ -f 01_GSE/GSE31210/GSE31210_pdata.csv \ -c "disease_state" \ --map "lung adenocarcinoma=Tumor" \ --map "adjacent normal=Normal" # 输出示例： # === Preview === # GSM773533 → Tumor # GSM773534 → Tumor # ... # GSM773758 → Normal # Total: 206 Tumor, 20 Normal

第三步：确认无误后加 -o 输出到文件

# 确认无误，加 -o 写入 groups.csv /media/share/secure/gse_download_and_cleaning/run_extract_groups.sh \ -f 01_GSE/GSE31210/GSE31210_pdata.csv \ -c "disease_state" \ --map "lung adenocarcinoma=Tumor" \ --map "adjacent normal=Normal" \ -o 01_GSE/GSE31210/groups.csv
常见场景与应对
场景pdata 示例处理方式
简单二分组 disease_state: "cancer" / "normal" --map "cancer=Tumor" --map "normal=Normal"
多分组需合并 "stage I" / "stage II" / "stage III" / "normal" --map "stage I=Tumor" --map "stage II=Tumor" --map "stage III=Tumor" --map "normal=Normal"
分组信息藏在描述中 "tissue: lung, status: cancer, age: 55" 换一列（用 -c 指定其他列）或联系负责人确认；不要自行猜测
无明确分组列 pdata 全是技术信息 标记为“分组信息缺失”，联系负责人补充
💡 关键原则：分组提取的核心是把每个样本映射为 Tumor/Normal（或 Treatment/Control 等）。始终先浏览（只 -f）再预览（加 -c --map）最后输出（加 -o），三步走，不要一步到位。如果分组信息不清晰或有多种分法，不要自行判断，联系负责人确认。

2.1 Bulk 数据格式校验

对每个下载的数据集执行以下检查，确认可进入后续分析：

2.2 单细胞数据下载与目录整理

单细胞数据可以用 run_download_gse.sh 的 --select-suppl 参数下载 GEO 附件，但脚本不会自动处理单细胞数据（不做探针映射、log2 检测等），下载后需要手动整理目录并用 R 构建 Seurat 对象。

2.2.1 下载原始文件
# 用 --select-suppl 模糊搜索并下载 GEO 附件（单细胞原始文件） # 参数值为模糊搜索关键词，匹配附件文件名，只能指定一次 /media/share/secure/gse_download_and_cleaning/run_download_gse.sh \ -g GSE184880 \ -o 01_GSE/GSE184880 \ --select-suppl "matrix" # 需要匹配多个关键词时，用正则 OR --select-suppl "snRNA|bulk"

下载后检查附件，常见三种格式：

  • 10X 三件套：barcodes.tsv.gz + features.tsv.gz（旧版 genes.tsv.gz） + matrix.mtx.gz
  • h5 文件：filtered_feature_bc_matrix.h5
  • 文本矩阵：*_counts.txt.gz

文件名通常带 GSM 前缀，如 GSM5601534_barcodes.tsv.gz。

2.2.2 目录整理（关键步骤）

下载后的文件通常平铺在同一目录，需按 GSM 分组整理到子目录，去掉文件名中的 GSM 前缀，旧版 genes.tsv.gz 重命名为 features.tsv.gz：

# 整理前（下载直接解压的平铺状态） GSE184880_raw/ ├── GSM5601534_barcodes.tsv.gz ├── GSM5601534_genes.tsv.gz # 旧版命名 ├── GSM5601534_matrix.mtx.gz ├── GSM5601535_barcodes.tsv.gz ├── GSM5601535_features.tsv.gz └── GSM5601535_matrix.mtx.gz # 整理后（按 GSM 分到子目录，去前缀，统一文件名） GSE184880_raw/ ├── GSM5601534/ │ ├── barcodes.tsv.gz │ ├── features.tsv.gz # genes.tsv.gz → features.tsv.gz │ └── matrix.mtx.gz └── GSM5601535/ ├── barcodes.tsv.gz ├── features.tsv.gz └── matrix.mtx.gz

整理要点：

  • 每个 GSM 单独一个子目录
  • 文件名去掉 GSM*_ 前缀
  • genes.tsv.gz 统一重命名为 features.tsv.gz（Seurat v5 的 Read10X() 只认 features）
  • h5 文件直接放入对应 GSM 子目录即可
💡 如何判断数据类型：如果 GEO 页面显示平台为 10x Chromium 或补充文件为“三件套”格式，则为单细胞数据。如果是 GPL570、HG-U133 等芯片平台或 Illumina HiSeq 并提供表达矩阵，则为 bulk 数据。

2.3 单细胞 Seurat 对象构建

目录整理完成后，将每个样本读入 R 构建 Seurat 对象。实际项目通常有多个 GSM 样本，需要逐一读取后合并。

2.3.1 读取数据（多格式兼容）
library(Seurat) library(data.table) name <- "GSE184880" if (!dir.exists(name)) dir.create(name) # 列出所有 GSM 子目录 sample_dirs <- list.files( paste0("../00_RawData/", name), pattern = "GSM\\d+", full.names = TRUE ) # 逐样本读取并建对象 scList <- list() for (i in seq_along(sample_dirs)) { project_name <- basename(sample_dirs[i]) if (dir.exists(sample_dirs[i])) { # 10X 三件套目录 counts <- Read10X(sample_dirs[i], gene.column = 2) } else if (grepl("txt.gz$", sample_dirs[i])) { # 文本计数矩阵 counts <- fread(sample_dirs[i], check.names = FALSE) counts <- tibble::column_to_rownames(counts, "V1") } obj <- CreateSeuratObject( counts = counts, min.cells = 3, min.features = 200, project = project_name ) # 关键：重命名 barcode 防冲突 scList[[i]] <- RenameCells(obj, add.cell.id = project_name) }
为什么要 RenameCells？不同样本可能有相同的 barcode，合并时会冲突。RenameCells(add.cell.id) 在 barcode 前加样本名前缀确保唯一。
2.3.2 合并多样本
# 确认全部成功 identical(length(sample_dirs), length(scList)) # 合并 scRNA_orig <- merge(scList[[1]], y = scList[2:length(scList)]) # 基本检查 table(scRNA_orig$orig.ident) length(colnames(scRNA_orig)) length(rownames(scRNA_orig))
2.3.3 QC 指标计算
# 线粒体（人 ^MT-，鼠 ^mt-） scRNA_orig[["percent.mt"]] <- PercentageFeatureSet(scRNA_orig, pattern = "^MT-") # 血红蛋白 scRNA_orig[["percent.hb"]] <- PercentageFeatureSet(scRNA_orig, pattern = "^HB[^(P|E|S)]")
2.3.4 添加分组并保存
⚠️ group.csv 需要手动整理：单细胞数据的 group.csv 无法用 run_extract_groups.sh 自动生成。你需要从 GEO 页面或文献中查找每个 GSM 样本对应的分组信息，手动创建 group.csv（两列：sample 和 group），其中 sample 列的值必须与 GSM 子目录名一致。示例：
# group.csv 内容示例（手动整理） sample,group GSM5601534,Tumor GSM5601535,Tumor GSM5601536,Normal GSM5601537,Normal
group <- fread(paste0("../00_RawData/", name, ".group.csv")) smps <- stringr::str_extract(Cells(scRNA_orig), ".*(?=_)") group_label <- group$group[match(smps, group$sample)] names(group_label) <- Cells(scRNA_orig) scRNA_orig <- AddMetaData(scRNA_orig, group_label, col.name = "group") qs2::qs_save(scRNA_orig, file = paste0(name, "/scRNA_orig.", name, ".qs2"))

  4. 确认分组：在备注列写清楚哪些样本作为实验组、哪些作为对照组
  5. 提交复核：反馈表随数据文件一起交付
