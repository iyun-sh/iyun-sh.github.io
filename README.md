# 综合规范页使用说明

## 当前结构

当前目录中的综合页已经切换为外部依赖模式，不再将源文档合并为单文件内嵌页。

三份独立文档的职责分工如下：
- `生信开发模块文件与输出规范.html`：模块目录、命名规范、日志与结果追踪
- `代码保护指南.html`：代码保护边界、交付限制、保护策略与红线清单
- `代码生成与验证流程图.html`：代码生成、验证、审核与交付流程

相关文件如下：
- `生信开发模块文件与输出规范.html`：规范手册正文
- `代码保护指南.html`：代码保护与交付边界说明
- `代码生成与验证流程图.html`：流程手册正文
- `综合规范与流程图.html`：综合查看页，负责页签切换与嵌入展示
- `综合规范与流程图_bk.html`：综合查看页备份

## 维护方式

以后如需更新内容，直接修改源文件即可：
- 修改规范内容：更新 `生信开发模块文件与输出规范.html`
- 修改保护指南：更新 `代码保护指南.html`
- 修改流程图内容：更新 `代码生成与验证流程图.html`

综合页会通过 iframe 直接引用上述独立 HTML 文件，因此无需再做“重新合并”“Base64 内嵌”或“单文件重建”之类操作。

## 综合页约定

- `综合规范与流程图.html` 与 `综合规范与流程图_bk.html` 都应保持外部依赖模式
- 页面应保留以下能力：
  - “规范手册 / 保护指南 / 流程手册”页签切换
  - iframe 自适应高度
  - 可单独打开源文件
- 当前模式标记为：`<!-- SINGLE_FILE=false -->`

## 最小检查清单

每次修改后建议检查：
1. `综合规范与流程图.html` 可以正常打开
2. 可以在“规范手册 / 保护指南 / 流程手册”之间切换
3. `生信开发模块文件与输出规范.html` 的修改会直接在综合页中显示
4. `代码保护指南.html` 的修改会直接在综合页中显示
5. `代码生成与验证流程图.html` 的修改会直接在综合页中显示
6. 页面中文内容无乱码

## 说明

如果未来确实要恢复为单文件交付，需要单独重新设计交付方式；当前默认工作流应始终以“独立源文件 + 综合查看页”作为准则。
